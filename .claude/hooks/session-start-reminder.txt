接手任务前先 Skill 调 using-superpowers 判定该走哪个 skill(1% 可能适用也先调)。按场景:
- 实现新功能/修 bug/重构 -> test-driven-development
- 多步实现或需隔离上下文 -> subagent-driven-development
- 方案探索 -> brainstorming
- 查 bug -> systematic-debugging
- 提交前自审 -> requesting-code-review
- 不确定调哪个 -> using-superpowers

跳过 = 理性化。"这次不需要""直接干更快""太简单不用测"都是 Red Flags。
