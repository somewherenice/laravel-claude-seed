# 备份与恢复约定

> 备份不是为了备份,是为了能恢复。没演练过的备份等于没有。

## 备份什么

| 对象 | 方式 | 频率 | 保留 |
|---|---|---|---|
| 数据库 | Cloud 自动备份 / `mysqldump` / `spatie/laravel-backup` | 每日全量 + 增量(binlog/WAL) | 按 RPO,常见 7-30 天 |
| 上传文件(`storage/app/private`、`storage/app/public`) | 对象存储版本化 / rsync | 每日 | 同 DB |
| `.env` / 密钥 | 离线保管(密码管理器 / Cloud secrets),不在仓库 | 变更时 | 永久 |
| 代码 | git,不需单独备份 | 每次提交 | 历史全留 |

## 恢复

- **定期演练**(至少季度):从备份还原到隔离环境,验证可用。
- **RTO / RPO 按业务定**:对账类系统 RPO 可宽松(日备),交易类 RPO 近零(binlog 持续同步)。
- **恢复步骤**:
  1. 停应用(或切维护页)
  2. 还原最近可用备份到新 DB 实例(不动原库,防二次破坏)
  3. 回放增量(binlog/WAL)到故障前
  4. 验证数据完整性(行数 / 关键表抽样 / 业务级硬校验)
  5. 切流量到恢复实例
  6. 事后写 ADR 记录故障与恢复

## 灾备(按需)

- DB 主从:主故障切从,降级读
- 跨区:核心系统异地副本,整区故障可拉起
- 非核心内部系统:单区 + 每日备份即可,不必上灾备

## 不该有的

- 只备份不演练
- 备份和源在同一台机器 / 同一区(一起没)
- `.env` 进备份压缩包(密钥随备份扩散)
